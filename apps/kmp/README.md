# KMP client — Compose Multiplatform

The ticketing client in Kotlin Multiplatform with shared Compose UI. Business logic and
UI both live in `commonMain`; platform code is confined to the edges (secure storage,
TLS pinning, entry points).

## Status

The **framework-free core is implemented and verified**: the `Outcome` result type, the
typed `AppError` taxonomy, the `UiState` model, the domain models, the repository ports,
and the first use cases (idempotent reservation, payment order create, order reconciler)
live in `shared/src/commonMain`, with a test suite in `shared/src/commonTest`.

What is verified, and how: the whole `commonMain` tree compiles clean with the installed
`kotlinc` (Kotlin 2.3.20, exit 0, no warnings), and the logic was exercised on the JVM
through fake repositories — 17 behavioural checks covering input hardening, the
double-tap idempotency-key passthrough, the payment unknown-outcome mapping, the reconciler
branches, and the domain invariants, all passing.

What is **not** yet done: a full Gradle sync has not been run in this environment. JDK 25
is installed but Kotlin has no JDK 25 target yet (it falls back to JVM 24) and Gradle 9.x
warns under JDK 25, so a green Gradle build wants a JDK 24 toolchain and a first
dependency download — the build files are in place and pinned, but I am not claiming a
Gradle build I have not watched go green. The Gradle wrapper jar and `gradlew` scripts are
generated by `gradle wrapper` and are intentionally absent until first sync. Compose UI,
the data/HTTP adapter, and the state holders are the next steps.

Build files present: `settings.gradle.kts`, root `build.gradle.kts`, `shared/build.gradle.kts`,
`gradle/libs.versions.toml`, `gradle.properties`, `gradle/wrapper/gradle-wrapper.properties`.

## Structure

Two modules, splitting UI-free logic from Compose UI so the logic tests on the JVM without
a simulator:

```
:shared                       UI-free: core, domain, data. Targets jvm + android + ios.
  src/commonMain/kotlin/com/ticketinglabs/client/
    core/        Outcome result type, AppError taxonomy, Recovery, UiState   [done]
    domain/model port usecase   validated models, ports, use cases           [done]
    data/        Ktor adapter → gateway base URL, DTO → domain mapping        [next]
  src/commonTest/kotlin/...     use-case + domain tests keyed to /shared/scenarios [done]
  src/androidMain, src/iosMain  platform actuals (secure store, pinning)       [next]

:composeApp                   Compose Multiplatform UI (a later module)        [next]
    ui/          atoms → molecules → organisms → templates → pages
    state/       ViewModels + StateFlow
    di/          wiring, base URL injection
```

Previews will use the unified `@Preview` in `commonMain` of `:composeApp` so every
atom/molecule/organism renders in isolation across its states. See
[`docs/client-architecture.md`](../../docs/client-architecture.md).
